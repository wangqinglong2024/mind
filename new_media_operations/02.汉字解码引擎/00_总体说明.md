# 汉字解码引擎 · 总体说明

> 受《01.项目总体规划/项目原则》P1–P10 约束。
> 引擎定义：**任何越南人熟悉的词 → 悬念 → 汉字揭晓**。满足这个定义的一切都是本引擎的选题。
> 格式与机制固定（资产），题材是弹药（轮换）。本文件夹下 01–06 为六个子系列的执行方案。

**一句话定位**：解码越南人自己的语言人生——你的名字、你骂人的话、你唱的歌、你吃的东西，背后全是汉字。

**账号**：主账号。**转化承接**：App 内"查你的汉字名"功能（P7 闭环）。

## 六个子系列（按爆火概率排序，01 最高）

| # | 子系列 | 叠加触发器 | 首周期配比 |
|---|--------|-----------|-----------|
| 01 | 身份系（姓/名/生肖/家乡） | 社交货币 | 35% |
| 02 | 生活系（食物/口头禅/漂移词） | 惊讶 | 20% |
| 03 | 热点系（网络语/节日/时事词） | 时效 | 10% |
| 04 | 互动飞轮系（评论区点单） | 与我有关（点名） | 15% |
| 05 | 情感系（情话/歌词/称呼） | 情绪 | 10% |
| 06 | 俗语系（成语/谚语） | 故事 | 10% |

排序依据：01 覆盖率碾压（阮姓一条命中 40% 观众）；02 惊讶钩最强且常青；03 蹭热点自带流量加权；04 单条爆发力极强但依赖评论区存量，冷启动期转不动，故居中；05 人群偏窄；06 故事结构 3 秒钩最难。注意配比 ≠ 排序——04 飞轮一旦转起来配比应上调，以周期数据为准。

引擎标配触发器：与我有关 + 好奇心缺口（含义后置）；各子系列再叠加一个，保证每条 ≥ 3 个（P3 超额达标）。

## 引擎级规范（六系共用，子文件不再重复）

- **结构**：0–1s 类别钩子上首帧 → 1–8s 悬念不给答案 → 最后 1 秒揭晓汉字（统一书法揭晓动画）→ 评论钩 + 回环。12–15 秒，硬上限 20 秒。
- **画面**：越南语大字幕（下 1/3）+ 汉字主角（大号居中）+ 小号拼音。禁英文（P2）。
- **人设三固定**（P8）：一个北方腔 TTS 声音、一套字卡与揭晓动画、一句固定开场。定型后永不更换。
- **红线**（P4）：只讲含义/文化/词源，禁命理断言；禁政治、主权、历史战争、民族比较；脏词遮挡+消音。
- **CTA**：必须是本条内容的自然延伸（名字条→"查你的汉字名"，歌词条→"歌词词包"）。
- **生产管线**：子系列词库 → LLM 批量越南语脚本 → 固定 TTS → 模板合成。人工只做两件事：筛钩子、读数据（P6）。
- **词源纪律**：每条发布前双语词典交叉验证，词源存疑的选题直接弃用——被评论区语言学爱好者纠错是信任级损伤。

## 迭代规则（P9）

- 每天 2 条，两周 30 条为一周期，按上表配比首发；此后配比完全由**子系列级 3 秒留存中位数**重定。
- 淘汰/加仓：单条 3 秒留存 < 60% 淘汰该钩子句式；> 70% 该变体立即做 3 个衍生。子系列连续两周期垫底 → 停产休整（词库不删）。
- 引擎级红线：整体 3 秒留存中位数 < 55% 连续两周期 → 回《项目原则》重新推导。
- 看板：3 秒留存 → 完播 → 评论率 → 主页访问 → 装机成本。播放量不进决策。
