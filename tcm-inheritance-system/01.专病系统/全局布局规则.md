# 全局布局规则

> 适用于管理端配置结果的渲染，以及应用端填报的展示。核心原则：**不支持可视化拖拽编排**，一切按"配置顺序 + 宽度"自动流式排布。

---

## 一、层级结构

```
表单
└── 页（分页，一次只显示一页）
    └── 卡片（页内纵向堆叠，各自整行）
        └── 字段（按宽度在卡片内流式排成若干行）
```

---

## 二、页（分页）布局与切换

- **一次只显示一页**，页与页之间通过翻页切换。
- **顶部**：步骤指示器，显示所有页标题与当前进度（如 `基本信息 › 辨证处方 › 随访　第 2/3 页`）。
- **底部**：`上一步` / `下一步`；最后一页显示 `提交`。
- **校验时机**：点 `下一步` 时校验**当前页**，未通过则停留并定位到出错字段；`提交` 时做**全表**校验。
- **页顺序** = 管理端配置顺序。
- 示例"3 页 2 卡片"：第 1 页含卡片A、卡片B（纵向叠放）；点下一步进第 2 页，以此类推。

---

## 三、卡片布局

- 卡片**必须整行（全屏宽）显示**，**不支持半屏、不支持两张卡片并排**——无论 Web 还是移动端都如此。
- 卡片在页内**纵向堆叠**，卡片之间**不并排**。
- 卡片 = 标题栏 + 内容区，可折叠。
- 卡片顺序 = 配置顺序，卡片间留统一间距。
- 字段的流式排布**只发生在卡片内部**（不会跨卡片）。

---

## 四、字段宽度与行排布（核心算法）

每个字段有一个**栅格宽度**（占卡片一行的比例）：`整行 / ½ / ⅓ / ⅔ / ¼ / ¾`。
系统按下面的规则自动排布，**无需也不支持手动摆放**：

**排布算法：**
1. 字段按**配置顺序**从左到右排入当前行。
2. 维护"当前行已用宽度"。放入下一个字段时：
   - 若 `已用宽度 + 本字段宽度 ≤ 整行`：放入**当前行**。
   - 否则：**换到新行**放本字段；同时把**上一行的最后一个字段拉伸**，吃掉该行右侧剩余宽度（消除空隙）。
3. **每个卡片的第一个字段**总是新行、**靠左**开始。
4. 同理，**卡片最后一行的最后一个字段**也拉伸填满，保证每一行右边缘都齐平。

> 结果：即便宽度组合不能整除，每行右侧也永远是齐的，无需可视化编排。

**图示（½ ¼ 换行的情形）：**
```
配置：f1=½  f2=¼  f3=½
┌───────────────┬───────────────┐
│   f1  (½)     │   f2 (¼→拉伸½) │   ← f3 放不下，f2 拉伸填满本行
├───────────────┴───────────────┤
│   f3  (½→拉伸整行)             │   ← f3 换到新行，作为末行最后一个也填满
└───────────────────────────────┘
```

**你给的例子：f1=½、f2=¼、f3=⅕**
- f1 → 本行已用 ½
- f2 → ½ + ¼ = ¾ ≤ 整行，放同行，已用 ¾
- f3 → ¾ + ⅕ ≈ 0.95 ≤ 整行，仍放同行；f3 是本行最后一个 → 拉伸吃掉剩余约 0.05
- **结果：三个字段在同一行**（f1 ½、f2 ¼、f3 拉伸约 ¼），行右侧齐平。

> 注：为保证整齐，宽度**仅从 `整行/½/⅓/⅔/¼/¾` 中选**，不提供 ⅕ 这类刻度（上例中的 ⅕ 会取最接近的 ¼）。任意组合都由"末尾填满"兜底，不会出现残缺行。

## 五、组件宽度限制（对《字段组件配置》的统一约束）

| 组件 | 可否设置宽度 | 说明 |
|---|---|---|
| 单行文本 | ✅ 可设 | 默认整行 |
| 单选（下拉） | ✅ 可设 | 统一下拉，宽度稳定 |
| 多选（下拉） | ✅ 可设 | 统一下拉，宽度稳定 |
| 数字 | ✅ 可设 | 短，默认 ½，适合并排 |
| 日期 | ✅ 可设 | 短，默认 ½，适合并排 |
| **多行文本** | ❌ 禁止 | 块级、可能多行，**固定整行** |
| **标签文本** | ❌ 禁止 | 标签随录入增多动态换行增高，**固定整行** |
| **图片** | ❌ 禁止 | 缩略图网格、块级，**固定整行** |
| **列表（表格）** | ❌ 禁止 | 表格、块级，**固定整行** |

> 规则：**凡是"多行/块级/高度不定"的组件（多行文本、标签文本、图片、列表）一律独占整行、不提供宽度设置**。它们参与排布时，会强制另起新行，且其后的字段也从新行开始。

## 六、响应式断点与移动端自适应

> **配置以 Web 为基准**（管理端按 Web 一行几个字段来配）。移动端不是简单地全部整行堆叠，而是按下面的**降档规则**自动重排。

### 断点
| 端 | 屏宽 | 排布方式 |
|---|---|---|
| 桌面 Web | ≥ 768px | 按配置的栅格宽度原样渲染 |
| 移动端 H5 | < 768px | 按下方"降档映射"自动重排 |

> 平板（768–1024px）默认沿用桌面规则；若后续需要更细，可加中间档，暂不做。

### 降档映射（Web 宽度 → 移动端宽度）
| Web 宽度 | Web 每行个数 | 移动端宽度 | 移动端每行个数 |
|---|---|---|---|
| 整行 | 1 | 整行 | 1 |
| ¾ | — | 整行 | 1 |
| ⅔ | — | 整行 | 1 |
| ½ | 2 | ½ | 2 |
| ⅓ | 3 | ½ | 2 |
| ¼ | 4 | ½ | 2 |

**一句话原则：移动端最多一行两列——凡 ≤ ½ 的字段统一变 ½，> ½ 的变整行。**

**示例（你的场景）：** Web 一行 4 个字段（各 ¼）→ 移动端每个变 ½ → 一行 2 个 → **4 个字段自适应为 2 行**。

### 其他约定（各端一致）
- **末尾填满**规则在移动端同样生效（每行最后一个字段拉伸填满剩余）。
- **块级组件**（多行文本、图片、子表）在任何端都**整行**。
- **卡片**在任何端都**整行、纵向堆叠**；**页**在任何端都照常**分页**。
- **子表**在移动端：**横向滚动表格** 或 **行卡片化**（每行折叠成小卡）二选一。

---

## 七、间距与对齐（基础规范）

- 卡片内字段：行与行、同行字段之间保持统一间距。
- 每行内的多个字段**顶部对齐**（标签在顶部时，输入框上沿对齐）。
- 卡片标题、字段标签、必填星号样式全局统一。
- Web 端卡片有最大内容宽度，超宽屏居中，避免一行拉太长。
