# 专病系统 · 动态表单模块 · 最终方案

> 名老中医传承系统下的「表单/字模板」模块。以下为一期完整方案，全部实现，无删减、无二期。

---

## 一、模块本质

一个**动态表单配置器**：
- **管理端**：医院自己配置要采集什么（表单、字段、选项、校验、分节、逻辑、权限），配好后可**预览**。
- **应用端**：用户按配置好的表单**填报**，支持 **H5 + Web 响应式**。

系统**不预置任何固定字典/标准词表**，一切采集内容由医院配置。定位是**传承数据库（可分析）**，"可分析"依靠**子表把数据拆成列**（如药名一列、剂量一列）来实现。

---

## 二、字段类型（8 种，全部实现）

| # | 类型 | 说明 | 场景举例 |
|---|---|---|---|
| 1 | **单行文本** | 单行输入 | 患者姓名、方名、诊断名称 |
| 2 | **多行文本** | 多行输入 | 主诉、现病史、刻下症 |
| 3 | **单选** | 单选，选项医院自配 | 性别、证型、疗效评级 |
| 4 | **多选** | 多选，选项医院自配 | 症状、兼证、合并用药 |
| 5 | **数字** | 数值输入，可设范围 | 年龄、剂量、疗程、化验值 |
| 6 | **日期** | 日期/日期时间 | 就诊、初诊、复诊、随访 |
| 7 | **图片** | 图片上传 | 舌象、面象、检查报告、处方影印 |
| 8 | **子表（表格）** | 容器，可配多列，应用端可加多行 | 处方（多味药）、多次随访 |

**子表规则：**
- 一个子表下可配置**多列**，每一列的类型是上面 **1~7 中的任意一种**（子表内不再嵌套子表）。
- 例：处方子表 → 列1 药名(单行文本/单选)、列2 剂量(数字)、列3 单位(单选)、列4 炮制(单行文本)。
- 应用端填报时，子表以**表格**呈现，用户可**加一行 / 删一行**。加行时表格**向下延展**（不分页），行多时随整页滚动，表头吸顶保证列名不丢。

---

## 三、字段可配属性（管理端配置项）

**所有字段通用：**
- 字段名（标签）
- 是否必填
- 提示说明（录入指引）
- 默认值

**校验规则：**
- 数字：最小值 / 最大值（如年龄 0–120）
- 文本：长度限制、格式校验（手机号 / 身份证 / 邮箱 / 自定义正则）
- 防空项（必填即非空校验）

**按类型附加：**
- 单选 / 多选：选项列表（增删改、排序）
- 图片：数量上限、单张大小上限
- 子表：列定义（每列 = 标签 + 类型 + 必填 + 选项/校验）、是否允许加行、最多行数

## 四、表单级功能（全部实现）

1. **页 / 卡片 / 字段（三层布局）**
   长病历分段录入，结构为 **表单 → 页 → 卡片 → 字段**：
   - **页**：分页展示，当前页只显示本页内容，可上一页/下一页切换（如 第1页 基本信息、第2页 辨证处方、第3页 随访）。
   - **卡片**：一页内可放多个卡片用于分组（如"基本信息"卡、"四诊"卡），卡片有标题、可折叠。
   - **字段**：卡片内放具体字段（子表也是卡片内的一个字段）。

2. **条件逻辑 / 显示跳转**
   按已填值控制显隐：如选"已手术"才显示手术相关字段；选"女"才显示孕产字段。规则形式：`当 [字段] [等于/包含/不等于] [值] 时，显示/隐藏 [目标字段 / 卡片 / 页]`。

3. **必填 + 校验规则**
   见第三节。提交时统一校验，未过则定位到出错字段。

4. **默认值 / 提示说明**
   常用值默认预填，字段旁显示录入指引。

5. **草稿保存 / 断点续填**
   填报中可存草稿，下次进入接着填，防止长表单丢失。

6. **数据脱敏 / 隐私**
   敏感字段（姓名、身份证、手机号）按**角色**决定明文 / 脱敏（如 `张*` `110***********1234`）显示。

7. **权限（角色）**
   角色分 **管理员 / 专家 / 录入员**，控制：可见的类目与表单、字段的查看与编辑权、脱敏级别。

## 五、类目管理

多级分类树 + 表单模板挂载：

```
类目树（可多级）
├── 专病（= 医案）
│   ├── 冠心病        → 挂 N 个表单（初诊表 / 随访表 / 疗效评价表）
│   └── 糖尿病
├── 专家数据          → 专家档案表单
├── 学术流派          → 学术思想 / 特色技法 / 优势病种 / 特色方药
└── 方剂库            → 方剂表单（含药味子表）
```

要点：
1. **一个类目下可挂多个表单模板**（非一类目一表）。
2. 类目可配：名称、图标、排序、启停、可见权限。
3. 表单与类目**解耦**，同一模板可在多个类目复用（如"疗效评价表"通用）。

## 六、交付物

1. **管理端配置器**：添加字段、配置属性、分节、条件逻辑、类目挂载，配好后**一键预览**。
2. **应用端填报**：按配置渲染表单，子表加行、草稿续填、校验提交；**H5 + Web 响应式**同一套自适应。
