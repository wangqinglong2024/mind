# 需求 · 通用规则与约束

> 跨模块的校验、脱敏、权限、草稿、空态与异常规则。

---

## 必填与校验
- **必填**：字段设为必填时，值为空（含未选、无标签、无图片、列表无行）判为不通过。
- **数字校验**：越过最小/最大值、超小数位数时不通过并提示。
- **文本格式校验**：手机号 / 身份证 / 邮箱 / 自定义正则，不符时提示。
- **列表校验**：受最少/最多行数约束，列可各自必填。
- **校验反馈**：出错字段就地标红并显示错误文案；分页表单校验定位到出错所在页。

## 条件显隐
- 规则：`当 [字段] [等于/包含/不等于] [值] → 显示/隐藏 [字段/卡片/页]`。
- 被隐藏的字段不参与其所在位置的校验与提交。
- 触发源常见于单选/多选/标签文本。

## 数据脱敏
- 敏感字段（如姓名、身份证、手机号）可按**角色**明文或脱敏展示。
- 脱敏示例：姓名 `张*`、身份证/手机号中间掩码。
- 脱敏作用于展示层（列表与查看），不改变已存储的真实值。

## 权限
- 角色：管理员 / 专家 / 录入员。
- 模板可配可编辑、可查看角色；字段可配可见/可改角色。
- 无权限的类目/模板对该角色不展示。

## 草稿与断点续填
- 长表单填报中可存草稿，再次进入可继续填写，避免丢失。

## 空态与提示
- 类目无模板：该类目不作为应用端 Tab。
- 模板无记录：列表显示空态并引导新增。
- 列表无行：显示空态提示引导添加。
- 所有确认/输入交互使用**系统自定义弹框**，不使用浏览器原生弹窗。

## 两端数据一致性
- 管理端保存模板/类目后，应用端按最新配置渲染。
- 应用端预览与实际填报使用同一渲染逻辑，效果一致。

## 明确不做
- 公式 / 查找引用（自动计算）。
- 评分 / 量表。
- 固定字典 / 预设标准词表（标签文本为纯自由输入）。

## 术语约定
- **模板** = 一张表单；**记录** = 基于模板的一条填报数据；**类目** = 专病/数据分类（**单级**，一个类目可挂多张模板）；**页/卡片/字段** = 模板内三层布局。
