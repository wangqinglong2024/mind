# 专病动态表单系统 · 需求文档 · 总览

> 名老中医传承系统下的「专病动态表单」模块。本套需求文档为最终完整版，逐模块描述功能与规则，配套原型见 `../原型设计/`，字段组件细则见 `../字段组件配置/`。

---

## 一、系统定位
一个**动态表单配置器**：医院自行配置要采集的内容（表单、字段、选项、校验、分页分组、类目），配置结果供采集人员填报。系统**不预置任何固定字典/标准词表**。最终目的是把专病医案等数据结构化，为后续的方剂筛选、用药习惯分析、学术流派数据库打好基础，因此强调**结构化、可统计、可关联**。

## 二、两个端
| 端 | 使用者 | 职责 |
|---|---|---|
| **管理端** | 管理员 | 配置类目、配置表单模板（页/卡片/字段/属性）、预览 |
| **应用端** | 录入员 / 专家 | 按类目 Tab 查看数据列表、新增填报、查看记录 |

两端通过统一的表单配置数据互通：管理端保存配置后，应用端即按该配置渲染列表与填报表单。

## 三、核心层级（术语）
```
类目（单级，不含子类目）
└── 模板（= 一张表单，一个类目可挂多张）
    └── 页（分页）
        └── 卡片（分组）
            └── 字段（9 种组件之一）
```
- **类目**：专病 / 数据的分类，**只有一级**（如 冠心病、糖尿病、专家数据），不支持子类目。
- **模板**：挂在类目下的一张表单，**一个类目可挂多张模板**（如 冠心病 下有 初诊表、随访表）。
- **页 / 卡片 / 字段**：模板内的三层布局结构。
- **记录**：应用端按某模板的一次填报，即一条数据记录。
- **应用端浏览路径**：选类目 → 选模板 → 查看该模板的记录列表。

## 四、字段类型（9 种）
| 类型 | 一句话 |
|---|---|
| 单行文本 | 短文本输入 |
| 多行文本 | 长文本输入（块级，独占整行） |
| 单选 | 从选项中选一个，**下拉呈现** |
| 多选 | 从选项中选多个，**多选下拉呈现** |
| 数字 | 数值输入，可设范围/单位 |
| 日期 | 日期/时间选择 |
| 标签文本 | 多个自由输入的标签，可按标签统计（块级，独占整行） |
| 图片 | 图片上传（块级，独占整行） |
| 列表 | 表格容器，可配多列、可加多行（块级，独占整行） |

> 每种类型的完整配置项见 `../字段组件配置/01~09`。

## 五、角色与权限
| 角色 | 权限 |
|---|---|
| 管理员 | 类目管理、模板配置、全部数据可见 |
| 录入员 | 按授权类目新增/查看记录 |
| 专家 | 按授权类目查看记录与统计 |
