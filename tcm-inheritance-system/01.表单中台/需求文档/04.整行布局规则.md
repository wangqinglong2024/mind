# 需求 · 布局与响应式规则

> 管理端预览与应用端填报共用同一套布局规则。完整版见 `../全局布局规则.md`。

---

## 层级
结构为 表单 → 页 → 卡片 → 字段；字段的流式排布只发生在**卡片内部**。

## 页
- 一次只显示一页，顶部页指示器，底部翻页。
- 「下一步」校验当前页，「提交」校验全表。

## 卡片
- 卡片**必须整行（全屏宽）**，不支持半屏、不支持两卡并排——任何端都如此。
- 卡片在页内纵向堆叠，含标题、可折叠。

## 字段宽度与行排布
- 每字段有栅格宽度：整行 / ½ / ⅓ / ⅔ / ¼ / ¾。
- **排布算法**（不支持可视化拖拽，纯按顺序 + 宽度自动流式）：
  1. 按配置顺序从左到右排入当前行；
  2. `当前行已用 + 本字段 ≤ 整行` → 放当前行，否则换新行；
  3. 换行时，**上一行最后一个字段拉伸填满该行剩余宽度**；
  4. 每卡片第一个字段总是新行靠左；
  5. 卡片最后一行的最后字段也拉伸填满 → **每行右边缘齐平**。
- **块级组件独占整行、禁止设宽度**：多行文本、标签文本、图片、列表；且其后字段强制另起新行。

## 响应式（配置以 Web 为基准）
- **断点**：≥768px 桌面（按配置渲染）；<768px 移动端（按降档映射）。
- **降档映射**：整行 / ¾ / ⅔ → 整行；½ / ⅓ / ¼ → ½。
- **一句话原则**：移动端最多一行两列——**≤½ 变 ½，>½ 变整行**。
  - 例：Web 一行 4 个（各 ¼）→ 移动端每个 ½ → 一行 2 个 → 4 个字段成 2 行。
- 末尾填满、卡片整行、页分页规则在移动端同样生效。
- 列表在移动端：横向滚动表格。

## 全屏宽度
- 管理端、应用端、预览、填报页均为**全屏宽度**，不设固定窄宽。
- 预览的移动端模式在全屏内居中显示手机宽度框（约 390px）。
