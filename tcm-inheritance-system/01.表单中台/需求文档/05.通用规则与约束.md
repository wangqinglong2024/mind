# 中台 · 通用规则与约束

> 跨模块的校验、删除、空态与异常规则。

---

## 一、必填与校验

1、**必填**：题目设为必填时，值为空（未选、未填、无标签、无图片、列表无行、富文本为空）判为不通过。
2、**数字校验**：越过最小/最大值、超小数位数时不通过并提示。
3、**列表校验**：列可各自必填。
4、**校验反馈**：出错题目就地标红并显示错误文案；分页表单定位到出错所在页。
5、搭建器保存/预览的前端校验见《01.表单搭建器·九》。

## 二、删除（硬删除）

1、**草稿删除**：删除该草稿。
2、**已下架 · 删除数据**：**彻底删除**该表单及其采集到的全部数据，不可恢复（区别于旧版"软删"）。
3、**隐藏数据 / 显示数据**：仅控制应用端数据列表是否展示该表单的数据，不删除数据。

## 三、发布与再发布

1、状态：草稿 → 发布 → 已发布；已发布 → 下架 → 已下架。
2、**发布后不可再编辑内容**（进入搭建器只读）。
3、**下架后禁止重新发布**。

## 四、空态与提示

1、列表无数据：显示空态并引导新增。
2、列表题无行：显示空态提示引导添加。
3、所有确认/输入交互使用**系统自定义弹框**，不使用浏览器原生弹窗（应用端原型部分为简化用了原生 confirm，正式实现统一自定义弹框）。

## 五、两端数据一致性

1、中台保存表单后，应用端按最新配置渲染。
2、预览与实际 查看/编辑/新增使用**同一渲染逻辑**，效果一致。
3、两端通过共享存储互通（原型用 localStorage 模拟）。

## 六、明确不做

- 栅格宽度：不做（全部整行）。
- 条件显隐/跳转逻辑：本期不做。
- 公式/自动计算、量表/NPS：不做。

## 七、术语约定

- **表单** = 中台配置的一张表单（业务侧可命名为"问诊单"等）。
- **分页 / 卡片 / 题目** = 表单内三层结构。
- **记录** = 应用端基于某表单的一条填报数据。
- **发布状态 / 类目** = **业务侧**概念，不属于中台。
- 注：旧版的"投放"改称"**发布**"，"适用医生"概念已**取消**。
