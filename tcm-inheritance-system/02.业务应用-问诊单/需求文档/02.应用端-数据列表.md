# 业务 · 应用端数据列表

> 录入员/医生查看某表单收集到的数据记录，并可直接在此**新增采集**。核心是把中台配置的"查询条件"落地为筛选、"在列表中显示"落地为列。
> 对应原型 `原型设计/app`。

---

## 一、进入路径（顶部二级导航）

1、页面顶部为**二级导航**：
   - 一级 = **类目**（标签页样式，下划线选中）。
   - 二级 = 该类目下的**表单**（胶囊样式）。
2、**只展示可见表单**：状态为 **已发布**，或 **已下架且"显示数据"** 的表单；草稿、已下架且"隐藏数据"的表单**不出现**。
3、**空类目隐藏**：某类目下没有可见表单时，该类目不在导航中出现；全部为空时提示"暂无已发布表单"。
4、导航中**不显示发布状态**（状态是搭建端的事）。
5、选中某表单 → 展示该表单的**记录列表**。

---

## 二、查询条件（右侧抽屉）

1、点【🔍 查询条件】从**右侧滑出抽屉**，每个条件**独占一行**；底部按钮 = 【清空】【查询】。
2、条件来自中台被设为"查询条件"的题目，按查询形式生成控件：

   | 查询形式（题型） | 控件 |
   |---|---|
   | 模糊查询（单行/多行/富文本/标签/列表） | 关键词输入框 |
   | 精准查询（同上） | 输入框（精确相等） |
   | 数字范围（数字） | 最小值 ~ 最大值 |
   | 日期时间范围（日期） | 起止日期 |
   | 单选查询（选择题） | **下拉**单选 |
   | 多选查询（选择题） | **下拉**多选 |

   注：单选查询、多选查询均以**下拉**呈现（多选下拉可勾多个）。
3、多个条件为 **AND** 组合；【查询】应用并关闭抽屉，【清空】清空全部条件并刷新列表。
4、**传给算法**：被设查询条件的字段标识 + 当前查询规则/取值，可打包传给算法接口做进一步筛选/计算。

---

## 三、列表

1、**列** = 中台勾选"在列表中显示"的字段 + 固定列（填写时间、操作）。
   - 可作为列的题型仅：**单选 / 单选打分 / 单行文本 / 数字 / 日期**；多选、多选打分、多行文本、标签、列表、富文本、图片**不支持**作为列。
2、**排序**：按填写时间倒序（默认）。
3、**每行操作**：查看 / 编辑 / 删除（见《03.应用端-填报与查看》）。
4、右上角【＋ 新增数据】：**直接在数据列表进行采集**（打开填报页录入一条新记录）。
5、空态：无数据时提示"暂无数据，点右上角「新增数据」录入"。
