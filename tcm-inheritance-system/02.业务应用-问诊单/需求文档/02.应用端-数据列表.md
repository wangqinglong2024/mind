# 业务 · 应用端数据列表

> 录入员/医生查看某问诊单收集到的数据记录。核心是把中台配置的"查询条件"落地为列表筛选。

---

## 一、进入路径

按业务分类/入口选中某问诊单 → 查看该问诊单的**记录列表**。
注：分类是业务侧自己的一套东西，与中台解耦。

---

## 二、查询条件（来自中台配置）

1、中台里被设为"查询条件"的题目，在**列表上方生成对应筛选控件**：
   | 查询形式 | 控件 |
   |---|---|
   | 文本·模糊匹配 | 关键词输入框 |
   | 文本·精准匹配 | 输入框（精确） |
   | 数字·范围 | 最小值 ~ 最大值 |
   | 日期时间·范围 | 起止日期时间 |
   | 单选 | 下拉筛选 |
   | 多选 | 多选筛选 |
2、多个条件为 **AND** 组合；支持重置。
3、**传给算法**：被设查询条件的字段标识 + 当前查询规则/取值，可打包传给算法接口做进一步筛选/计算。

---

## 三、列表

1、列 = 中台里勾选"在列表中显示为列"的字段 + 固定列（填写人/填写时间/状态）。
2、**排序**：按填写时间倒序（默认）。
3、点击某行 → 查看该条填报记录（见《03.应用端-填报与查看》）。
4、空态：无数据时显示空态引导。
